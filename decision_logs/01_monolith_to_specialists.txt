DL-01 — Monolith to specialists: does decomposition actually improve output?  ✦
Marketing Agent POC


0. Context

This is the decision the whole project hinges on. The assignment asks whether
specialized agents produce better output than one generalist agent. I built the
monolith first to have a real comparison, not a theoretical one.


1. Research question

Does breaking the pipeline into specialist agents produce meaningfully better
campaign output than one agent doing everything — and where does the improvement
actually come from?


2. Current LO stage

[x] Analyzing  — A1 research: agencies specialize because different tasks need
                  different context, not because of org structure preferences
[x] Designing  — mapped agency roles to agent scope: product analyst, account
                  planner, creative, legal reviewer, media strategist
[x] Realizing  — built the monolith first, compared output side-by-side,
                  added audience as a dedicated middle stage to prove the point


3. What makes a good decision here

- The comparison is fair: same model, same image, same description
- "Better" is observable without asking the model to evaluate itself
- The audience stage adds value rather than just adding latency


4. What I decided

Decompose by agency role. Five agents, each with a narrow scope and only the
context it needs. The audience agent sits between vision and copywriter —
its job is to turn product signals into a persona the copywriter writes for.


5. Why this decision

Method: Showroom + Lab

The monolith prompt: "You are a marketing expert. Analyze this product image
and write two ad variants." Output on the eco water bottle:

  "Check out this amazing product! Stay hydrated and save the planet. Shop now!"
  Audience: general public

The problem is the prompt asks the model to do product analysis, audience
inference, copy tone, and CTA generation simultaneously. With no separation,
each task degrades the others. "General public" is not an audience.

The A1 research gave the insight: agencies don't have one person do everything
because the context requirements are different. The creative team that also
defines the audience defines it in terms of who they imagine wanting the copy
they've already mentally written. Separating the stages forces the audience to
be derived from product signals, not copy assumptions.

With the multi-agent pipeline on the same input:

  Audience: "Eco-Conscious Millennial Athlete, 25-34, sustainability + fitness"
  Variant 1: "Stay hydrated, stay green. 24h cold, zero plastic compromise. #LiveGreen"
  Variant 2: "Insulated. BPA-free. Built for your gym bag and the planet. Shop the change."

Same model. Same image. The difference is that the copywriter received a
pre-built persona instead of raw product tags. It had one job.

Adding the audience stage was the concrete proof. Without it, the copywriter
received vision tags directly and produced better-than-monolith but still
generic output. With it, copy was persona-specific.

What I'd recommend:
Build the monolith first. Running it is faster than arguing about whether you need
specialists. The output quality gap makes the case in one comparison.

GAP found:
The copywriter prompt doesn't constrain the variant angles — both variants
could use the same structure. Added: Variant 1 must open with an emotional hook,
Variant 2 must open with a product benefit. Without this, specialization improves
targeting but not differentiation.


6. Does this hold up

- Output improvement observable: ✅ "general public" → "Eco-Conscious Millennial"
                                     is the clearest possible signal
- Audience stage adds value:      ✅ without it, copy is better but still generic;
                                     with it, copy references persona identity
- Latency acceptable:             ✅ audience agent adds ~20s; output quality gain
                                     justifies it

Assumptions I'm making:
The improvement generalises beyond the water bottle test case. I ran it on two
products (water bottle, bamboo toothbrush) — both showed the same pattern.

What surprised me:
The specialization improvement isn't about model capability — it's about
information architecture. The copywriter was given a narrower scope and the
right context. Same model, same weights.


7. What this unlocks

Next LO stage: Realizing → Realizing (remaining agents — compliance, CTA,
RAG, refinement)

What I can now do:
Add the remaining agents on top of a validated audience → copy pipeline.
Each new stage has a clear input and output contract.

Ongoing monitoring:
If copy starts coming back generic again, check the audience output first —
a weak persona is almost always the root cause.

How I'll know this worked:
persona_label is never "general audience" or "everyone" on any product-specific
input. Variant 1 and Variant 2 are structurally different on first read.
